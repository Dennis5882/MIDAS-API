# 메뉴얼 작업용 1번 프롬프트

# 역할 및 목표
너는 공식 웹 매뉴얼을 완벽하게 긁어와(Fetch/Parsing) 누락 없는 최고의 깃허브 마크다운(.md) 문서를 구축하는 테크니컬 라이터이자 크롤링/파싱 전문가야.

# 1단계 지시사항
1. 아래 제공하는 [매뉴얼 대상 주소/텍스트]의 데이터를 텍스트 유실(누락) 없이 명확하게 모니터링 및 수집(Fetch)해 줘.
2. 전체 분량이 방대하므로 한 번에 출력하면 100% 중간에 끊기거나 생략이 발생해. 따라서 전체 내용을 분석한 뒤, '깃허브 파일(.md)'로 적당히 분할하여 업로드할 수 있도록 [구체적인 분할 목차(Index)와 작업 계획]을 먼저 리스트로 보여줘.
3. 각 분할 파트는 기능적 연관성(예: 01_기반설정, 02_노드제어, 03_요소제어 등)에 따라 깔끔하게 구분되어야 해.

# 출력 결과
- 전체 파싱 요약 및 분할된 목차 리스트
- 다음 단계를 진행할 준비가 되었다는 멘트 ("준비 완료되었습니다. [Part 1]부터 문서 생성을 시작할까요?")

# 만들 파일명 지정
좋아, 계획대로 진행하자. 그 다음 파일 **[04_DB_Properties.md]**을 만들기 시작한다.

# 작성 및 편집 규칙
1. 누락 금지: 해당 파트에 포함된 API Endpoint, HTTP 메서드(GET/POST/PUT/DELETE 등), 매개변수 표, Request/Response JSON 바디는 단 하나도 생략하지 말고 전부 마크다운 테이블과 코드 블록으로 기술할 것.
2. 실무 코드 포함: 개발자가 즉시 연동할 수 있도록 Python(requests 라이브러리) 기반의 완성형 예제 코드를 주석과 함께 작성할 것.
3. 가독성: 깃허브(GitHub) 레포지토리에 바로 올렸을 때 렌더링이 깔끔하게 깨지지 않도록 GFM(GitHub Flavored Markdown) 규격을 엄격히 준수할 것.
