MIDAS Gen NX MAPI — Auto Save Before Analysis

KO해석 실행 전 파일 자동 저장 패턴  |  ENAuto-save pattern before running structural analysis

Overview

KO  MIDAS Gen NX의 MAPI는 해석(POST /doc/anal)을 실행하기 전에 반드시 파일을 먼저 저장(POST /doc/saveas)해야 합니다. 저장 없이 해석하면 Gen NX가 저장 다이얼로그를 띄워 자동화가 중단됩니다.

이 예제는 GET /mapikey/verify로 현재 로그인된 사용자의 이메일을 가져와 저장 경로를 동적으로 만들고, POST /doc/saveas → POST /doc/anal 순서로 실행하는 전체 흐름을 보여줍니다.
EN  MIDAS Gen NX MAPI requires saving the file (POST /doc/saveas) before running analysis (POST /doc/anal). Without saving first, Gen NX shows a save dialog that interrupts automation.

This example retrieves the logged-in user's email via GET /mapikey/verify, builds a dynamic save path from it, then runs POST /doc/saveas → POST /doc/anal in the correct order.

Flow / 동작 흐름

1GET /mapikey/verify → j.user (e.g. "john@midasit.com")
↓
2extract username → "john"  ·  build path C:/Users/john/Documents/gen_rc_temp_1234567.mgbx
↓
3POST /doc/saveas  { Argument: "<path>" }
↓
4POST /doc/anal    {}

Why Order Matters / 순서가 중요한 이유

⚠️ KO  /doc/anal을 /doc/saveas 보다 먼저 호출하면 Gen NX가 "저장하시겠습니까?" 다이얼로그를 표시하고 응답을 기다립니다. MAPI 호출은 타임아웃 또는 에러 없이 무한 대기 상태가 될 수 있습니다.

⚠️ EN  Calling /doc/anal before /doc/saveas causes Gen NX to show a "Do you want to save?" dialog and wait for user input. The MAPI call may hang indefinitely without a timeout or error.

Unit Setting Before Build / 단위계 선제 설정

⚠️ KO  모델을 만들기 전(PUT /db/node 호출 전)에 반드시 단위계를 설정하세요. Gen NX가 kN/m 등 다른 단위로 열려 있으면 좌표 7000(mm 의도)이 7000 m로 해석되어 해석 결과에 이상 변위 경고가 발생합니다.

⚠️ EN  Always set the unit system before creating model entities (PUT /db/node). If Gen NX is open in a different unit (e.g. kN/m), the coordinate 7000 (intended as mm) is interpreted as 7000 m, causing abnormal displacement warnings.

Code / 코드

/**
 * KO: Base URL에서 /gen, /civil 경로를 제거해 루트 URL 반환
 *     /mapikey/verify 는 루트(포트 바로 뒤)에 위치합니다.
 * EN: Strip /gen or /civil suffix to get the root URL.
 *     /mapikey/verify lives at the root, not under /gen or /civil.
 */
function mapiRoot(base) {
  return base.replace(/\/(gen|civil)\/?$/i, '');
}

/**
 * KO: 연결 확인 및 로그인 사용자 이메일 반환
 *     j.user 에 이메일 전체가 담깁니다 (예: "john@midasit.com")
 * EN: Verify connection and return the logged-in user's email.
 *     j.user contains the full email (e.g. "john@midasit.com")
 */
async function verifyConn(base, key) {
  const res = await fetch(`${mapiRoot(base)}/mapikey/verify`, {
    headers: { 'MAPI-Key': key }
  });
  let j = {};
  try { j = await res.json(); } catch (e) {}
  return { ok: res.ok, j };
}

/**
 * KO: 저장 경로 자동 결정
 *     성공: C:/Users/{username}/Documents/gen_rc_temp_{timestamp}.mgbx
 *     폴백: C:/Temp/gen_rc_temp_{timestamp}.mgbx  (verifyConn 실패 시)
 *
 *     ※ %USERPROFILE% 같은 환경변수는 MAPI 서버(Gen NX 프로세스)가 인식하지 못합니다.
 *       /mapikey/verify 의 j.user 이메일로 사용자 폴더를 직접 구성해야 합니다.
 *
 * EN: Determine save path automatically.
 *     Primary : C:/Users/{username}/Documents/gen_rc_temp_{timestamp}.mgbx
 *     Fallback: C:/Temp/gen_rc_temp_{timestamp}.mgbx  (if verifyConn fails)
 *
 *     ※ Environment variables like %USERPROFILE% are NOT resolved by the MAPI server.
 *       Build the path from j.user returned by /mapikey/verify instead.
 */
async function autoSavePath(base, key) {
  try {
    const { ok, j } = await verifyConn(base, key);
    if (ok && j?.user) {
      const username = j.user.split('@')[0];           // "john@midasit.com" → "john"
      return `C:/Users/${username}/Documents/gen_rc_temp_${Date.now()}.mgbx`;
    }
  } catch (e) {}
  return `C:/Temp/gen_rc_temp_${Date.now()}.mgbx`;     // fallback
}

/**
 * KO: 단위 설정 → 모델 생성 → 저장 → 해석 전체 흐름
 *     setUnit 은 모델 생성(노드 추가) 전에 반드시 호출해야 합니다.
 * EN: Set unit → build model → save → analyze — full flow.
 *     setUnit MUST be called before creating any model entities (nodes/elements).
 */
async function runAnalysis(base, key) {
  const headers = { 'MAPI-Key': key, 'Content-Type': 'application/json' };
  const api = (method, path, body) =>
    fetch(`${base}${path}`, { method, headers, body: body ? JSON.stringify(body) : undefined });

  // ① 단위계 선제 설정 (KO: 노드 추가 전 필수 / EN: must call before PUT /db/node)
  await api('PUT', '/db/unit', { Assign: { 1: { FORCE: 'N', DIST: 'MM' } } });

  // ② ... 모델 생성 (nodes, elements, loads ...) ...

  // ③ 저장 — 반드시 /doc/anal 보다 먼저 (EN: must precede /doc/anal)
  const savePath = await autoSavePath(base, key);
  await api('POST', '/doc/saveas', { Argument: savePath });

  // ④ 해석 (EN: run analysis)
  await api('POST', '/doc/anal', {});

  return savePath;
}
